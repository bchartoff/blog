It's been a month (and three days) since I started my fellowship at The Washington Post. In a word, I've spent the month building. In a few more words I've been building things, building projects, and building skills.

Building Things

At the end of these ten months, I want to be able to look back and see concrete things that I made. Here are a few little projects I've built so far: Equalhalves: A few weeks ago, this map splitting the US GPD in half. It's got problems (short version, it's basically a population map, compare to this). In a chat about this map in the #opennews IRC room, Ted Hanoffhandedly mentioned it would be cool to see lines which split the population of the US in half. I thought it would be cool too, so I built a project that found them. It was a lot of fun. The quick and dirty version drew straight lines across the US. At this scale, there's a ton of data loss caused by the curvature of the Earth, so I decided to spend a day rebuilding the script to draw great circles instead of lines (the trig is fun and you should check it out). Githubchild: This was an afternoon's work (yesterday) that sprung from a twitter conversation with Dave Stanton a few days ago. He asked twitter:

Are there a @github web shortcuts to go next/prev through commit history?

— Dave Stanton (@gotoplanb) March 4, 2014
It turns out that github has some keyboard shortcuts, including "p" to view the parent commit of the commit you're currently viewing. Github doesn't have a "view child" shortcut, which makes sense for large, complex repos — traversing up (child → parent) the commit tree fundamentally converges (in general) while traversing down (parent → child) might lead down far flung divergent branches. That being said, it makes sense for relatively simple repos to be able to travel in both directions. I wrote this little function in javascript to press "c" to move parent → child. It's probably a work in progress, since it's a bit slow. Right now I'm calling to the github API once per page load — I should really speed things up by storing parent:child pairs in local storage, or making a call to a python based server. Soon. Crimescrape: This is a small scraper I built for Ted Mellnik, a database editor in the graphics department here at the Post. It pulls down DC crime statistics from the DC crime map by navigating a few forms, dropdowns, checkboxes, etc. I wrote it a while back, but just put it up on github a few days ago. While it's certainly single-use, many of the concepts might be useful to check out. In the README, I spelled out the entire process of hosting and running this script. For example, check out how I ran Firefox on a remote Ubuntu server, using a headless browser. Fun times. All told, these projects took maybe a week of my time so far. So what else have I been up to? Lots.

Building Projects

I've got a number of ongoing projects in the works, some of which will get their own blog posts down the line. Instead of another laundry list of projects, here are some lessons I've learned about how to Get Stuff Done as a fellow in a newsroom.

Get off the laptop

It's easy to get tunnel vision on a project, and code away in isolation all day. But (most of the time) that's not the best way to get stuff built. Almost all of the projects I've accrued have been pitched, developed, and/or fleshed out while I'm away from my desk. I go to meetings (so far with the Design team (specifically their "code" meetings with some of the most amazing folks I've ever had the pleasure of working with), the Politics desk, Local desk, and scattered other teams and folks), I go ask questions, or I just stand over people's shoulders and see what they're up to (it's not as creey as it sounds, I promise). It's worth mentioning that I've applied these same practices without actually leaving my desk (just getting off the metaphorical laptop, if you will). Two of the projects I mentioned above came from ideas on Twitter and IRC (and from sources outside of my own newsroom). The Post also makes great use of chat (specifically HipChat) for keeping teams in constant contact.
